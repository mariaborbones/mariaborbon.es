# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

María Borbonés's personal website — a plain HTML/CSS/JS static site with no build step and no dependencies.

## Commands

- Preview locally: open `index.html` directly in a browser, or run `npx serve .` for a local server.
- No install, build, lint, or test commands — there is no package.json or build tooling.

## Architecture

- `.site-header` is an app-style fixed top bar: `.logo` (left) and `.header-right` (right) — `.header-social` (inline SVG icons, `currentColor`, for LinkedIn/X/YouTube; update the `href`s directly in `index.html` if a handle changes) plus `.header-contact`. The whole header uses `mix-blend-mode: difference` on white text/icons so it stays legible over both the scene and the light sections below without per-section color overrides — keep new header children white (`color: #fff`) rather than fighting that trick with an explicit color. Below 480px, `.logo` and icon sizes/gaps shrink (see the header's own `@media` block) since three social icons + "Contacto" next to a big Caveat wordmark is tight on small phones.
- `index.html` — the hero is a GSAP ScrollTrigger horizontal "walking scene" (`.scene` → `#world`, pinned in place) telling María's day as 7 time-stamped zones (`zones` array in `main.js`, 06:00 gimnasio → 06:30 casa → 07:00 camino al trabajo → 07:30 Verti → 18:00 colegio → 18:15 súper → 18:45 en casa). `.character-group` holds `.mum-jump > .figure.mum` (always present, walking in a fixed screen position while `.zone` buildings scroll past underneath), `.kids-group` (boy + girl + stroller), and `.family-car` (shown only during the 07:00 commute zone — `DRIVING_ZONE` — while `.mum` is faded out, since she's driving not walking). The Verti zone (`.verti-office`, `.sign-verti`, `.verti-logo`) recreates Verti Seguros' real orange-to-red gradient wordmark in CSS (`background-clip: text`) rather than embedding their logo asset. After the pin releases, normal vertical `<section>`s follow directly: `.worklife` (`#worklife`), `#work` (Proyectos), and `#contact`. There used to be a plain `.evening` cards section (19:30/20:00/20:30) and a fixed "Mi día" stats HUD between the pin and those sections — both were removed (the HUD kept overlapping the sections below on narrower viewports); don't reintroduce a fixed-position (`position: fixed`, document-level) overlay or an extra section there without checking for that same overlap. The `.game-legend` "Cómo jugar" panel (top-right, explains the ❓ blocks) brings the stat bars back without that bug: it's `position: absolute` inside `.scene` (which is `position: relative` and the thing GSAP actually pins), not `position: fixed` on the page — so it rides along with the pinned scene and simply scrolls away with it once the pin releases, instead of needing its own show/hide ScrollTrigger. Inside it, `.game-stats` holds the four live bars (🔋 battery, 💪 health, 🧠 knowledge, ❤️ love — `.stat-track`/`.stat-fill`/`.stat-value`, same DOM shape as the old HUD), driven by `updateStats()` in `main.js`: called on every zone change (`zones[i].stats`, so battery keeps draining even without a reward box) and on every reward-box hit (`box.stats`, a jump to the *next* zone's numbers). Since evening cards no longer exist, there's no separate evening-stats wiring — the bars' entire lifecycle is the pinned scene.
- Four zones (gimnasio, Verti, colegio, súper) each have a static Mario-style `.question-block` positioned near that zone's exit, purely as a playful visual beat. `rewardBoxes` in `main.js` gives each of the 4 `.question-block`s a fixed world-position (`zoneIndex * ZONE_WIDTH + BOX_CENTER_OFFSET`, matching where the CSS actually places `.question-block` inside `.zone`), and `mumCenterX` is measured once (mum's screen position never changes — she's the one standing still while the world scrolls). Every frame, `checkRewardBoxes()` compares each box's current screen position (world position minus scroll distance) against `mumCenterX`; the instant a box reaches her, it fires once (`box.hit` flag — Mario blocks don't come back) and plays `triggerBonk()`: `.mum-jump` (a wrapper kept separate from `.mum` itself so this doesn't fight the walk-cycle's own bob tween) hops up into the block, the block squashes and turns "used" (`.hit` class), and a matching emoji (`.stat-popup`) floats up from the block and fades. The colegio box additionally reveals `.kids-group` — but that reveal is checked continuously (`checkKidsVisibility()`, reversible if you scroll back), separately from the one-shot bonk, since kids appearing/disappearing should track her actual walking position, not stay stuck on after a single trigger.
- `assets/css/style.css` — all visual design: the zone/building illustrations and the car (pure CSS shapes, no images), the character rig (`.figure .head/.body/.arm/.leg`, sized per `.girl`/`.boy`/`.mum`), the stroller, the sky's single dawn→day→dusk gradient spanning the whole world width, the question-block/popup (`.question-block`'s `left`/width in CSS must stay in sync with `BOX_CENTER_OFFSET`/`ZONE_WIDTH` in `main.js` — they're not derived from each other), and the plain-section styles below the scene. The "Caveat" handwriting webfont is the site's brand voice, used deliberately big and colorful wherever María's own personality/name should read as unmistakably hers: `.logo` (header), `.caption-title` (the big zone name in the scene, e.g. "Gimnasio"/"Colegio"), `.sign-label` (the small pill under each building — `.sign-gym` overrides its color back to white since that pill has a dark background), plus the `.worklife`/`.game-legend`/`.project-card` headings noted below. Keep new headline-ish text in that font/scale rather than plain system-ui if it's meant to feel like "her" voice.
- `assets/js/main.js` — GSAP setup: registers ScrollTrigger, pins `.scene` for a scroll distance equal to `world.scrollWidth - viewport width`, and on every scroll update drives (a) the world/sky/clouds/ground parallax via `backgroundPosition`/`x`, (b) a scroll-distance-driven walk cycle (leg/arm `rotate` via `Math.sin`, per-figure phase offset, stroller/car wheel spin), (c) the zone caption crossfade + progress bar, (d) `updateCharacters()`, which crossfades mum/car opacity based on which zone index is active, and (e) `checkKidsVisibility()`/`checkRewardBoxes()` (see above). GSAP + ScrollTrigger are loaded from the jsdelivr CDN (`<script>` tags in `index.html`), keeping the "no build step" approach — don't introduce a bundler/npm install to add JS dependencies here.
- `.worklife` (`#worklife`, directly after the pinned scene) is a whiteboard-sketch-style section grounded in María's public LinkedIn/professional profile: a 3×3 CSS Grid (`.worklife-grid`, `grid-template-areas`) with her real photo (`assets/img/maria-worklife.jpg`) in the center cell (not interactive) and 8 hand-drawn-style "sticky note" cards (`.wl-node`, one per grid area) around it — role at Verti, career history, education, values, colleagues (deliberately generic 👤 icons in `.wl-avatar`, never real photos of coworkers), public speaking, and a callback link to the day-in-life scene. Each `.wl-node` is collapsed by default, showing only `.wl-head` (icon + title + `.wl-chevron`); `setupWorklifeCards()` in `main.js` toggles an `.open` class on click/Enter/Space, which drives `.wl-body`'s `grid-template-rows: 0fr → 1fr` transition (the "animate to auto height" CSS grid trick — content lives in `.wl-body-inner` so the transition has something to clip). `.worklife-grid` uses `align-items: start` (not `stretch`) precisely because collapsed cards must hug their own header height instead of matching the tallest cell in their row (the center photo). Headings use the "Caveat" handwriting webfont (Google Fonts CDN, linked in `<head>`) for the whiteboard-marker feel; collapses to a single column below 860px. Playful touches layer on top of the toggle: `.wl-node:hover .wl-icon` runs the shared `wl-wiggle` keyframe, opening a card runs `wl-pop` on its icon, and `popWorklifeIcon()` in `main.js` spawns a one-off `.wl-pop-emoji` clone of the icon that floats up and fades via GSAP (reusing the same "reward popup" idea as `triggerBonk()`, just simpler — no timeline). Two small emoji doodles (`.worklife-board::before`/`::after`) decorate the board corners, hidden below 600px. Copy is intentionally written with some personality/humor (not dry bio text) — keep that tone if editing. Update the copy directly in `index.html` if her role/bio changes — it's not data-driven.
- `#work` (Proyectos personales, after `.worklife`) uses `.project-grid`/`.project-card` — an auto-fit card grid, each card getting a distinct accent color + slight rotation via `:nth-child(3n+1/2/3)` (matching the whiteboard cards' hand-drawn feel) and an icon that wiggles on hover (`wl-wiggle`, shared with `.worklife`). Currently 3 placeholder cards with intentionally playful/vague copy for María to swap for real projects as she publishes them.
- `render.yaml` — deploys the site as a Render static site (`staticPublishPath: .`, no build command). Pushing to the connected branch auto-deploys.

Keep this structure flat: add new pages as sibling `.html` files, avoid introducing a build step, and when tweaking the walking scene's CSS-shape illustrations, verify visually (e.g. Playwright screenshots at several scroll fractions) rather than trusting the CSS by inspection alone — the geometry is fiddly (nested `position: absolute` coordinate frames are a recurring source of bugs here) and looks nothing like the code until rendered.
